First Node.js Server - Hands on Node.js Tutorial

Coding a Node.js server is so simple that even plain language explanation can be complex when compared to the coding language. So, we will directly dive into the Code first, and then explain it (though it might not be neccessary anymore, trust me...)
In this tutorial I will assume Node.js is installed in your system. And you know how to move/navigate through different directories in command prompt / terminal.
Save the following code in a file and name it "server.js". 
                
                    //require the module name "http", and assign it to any variable for later use     
                    var http = require("http");

                    //now create a server and assign it to any variable for later use
                    var server = http.createServer();

                    //now give the server a port to listen to...
                    server.listen(9090);
"Modules" in node.js are just pre-written code files that we use to make our lives easier. Think of it like ".h" header files that we include in our C programs, or packages that we import in Java. Here, in node.js we use "require" in place of "include" or "import". That's it, that's all...
Technically your server is ready now to be run (But it is not complete yet...)
Go to command prompt in windows, or terminal in mac or linux. Navigate to the directory where your server.js file is saved. and type, 
                
                    node server
                    //yes you can omit the ".js" extension
Now, if you go to your browser and type "localhost:9090" and hit enter, you will see the request is loading for a long time, showing "waiting for localhost" and after a long time, it might just time out and stop... So, you might be asking yourself, did it worked? did the server run?

The answer is "Yes, it worked..."
The browser could reach the server and wait for its response. But we didn't code our server to do anything or send any response when a request comes in... So, let's fix it...
You can terminate/stop the server anytime by pressing Ctrl+C in your console.
We need to tell our server what it need to do when a request comes in. So, let's define a function to handle incoming requests. 
                

                    //this function will handle the incoming requests
                    function handleRqst (request, response) {
                        response.end("Congratulation... Your Node.js server is now up and running...");
                    }
Now, we need a way to tell node.js that whenever a reuqest comes in, call our function "handleRqst" to process the incoming request and do whatever we want it to do. Fortunately, we can pass a function as a callback function to the "http.createServer()" method we created earlier. So, let's modify our server.js file to look like this... 
                
                    //require the module name "http", and assign it to any variable for later use     
                    var http = require("http");

                    //now create a server and assign it to any variable for later use
                    var server = http.createServer(handleRqst);

                    //this function will handle the incoming requests
                    function handleRqst (request, response) {
                        response.end("Congratulation... Your Node.js server is now up and running...");
                    }

                    //now give the server a port to listen to...
                    server.listen(9090);

                    //let's leave a message for ourselves in console/terminal
                    console.log('Server is running at localhost:9090');
Now hit the browser with "http://localhost:9090/" and Bingo! you should see the message :
Congratulation... Your Node.js server is now up and running...
